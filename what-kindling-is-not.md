# What Kindling Is Not

Scope boundaries matter. This document exists to prevent misunderstanding — and to hold PUC accountable to these constraints.

---

## Not a closed marketplace

Every Kindling module is open-source, self-hostable, and bypassable. An agent or developer can:

- Use Igniter without registering with Verifier
- Build a competing router that ignores Scout entirely
- Run a local Verifier instance with custom methodology
- Bypass any module and interact with x402/MCP/A2A directly

No lock-in. No penalty for leaving. If Kindling stops being useful, agents and developers should stop using it.

---

## Not pay-to-rank

Referral split size has zero effect on ranking or routing position. A service offering a 50% referral split does not rank higher than a service offering 0%. This is enforced in code, not policy alone.

Paying more does not buy visibility, better scores, or routing preference. The only path to a better Verifier score is building a better service.

---

## Not hidden affiliate routing

Every referral relationship is disclosed:
- In 402 response headers, **before payment**
- In contract events, **after payment**
- In Igniter's README
- In Verifier metadata

There are no hidden kickbacks. Every split is on-chain and auditable by anyone.

---

## Not benchmark theater

Documenter reports follow this process, in order:
1. Methodology published before any results
2. Competitors included — not just the service being evaluated
3. Losses shown — if PUC's service performs worse on a dimension, that is published
4. Raw data published alongside every report

A Documenter report that excludes competitor advantages will not be published. A report that shows only wins is not a benchmark — it's marketing. We don't publish marketing as benchmarks.

---

## Not registry spam

MCP tool descriptions and A2A Agent Cards generated by Igniter are optimized for **accuracy and relevance**, not keyword stuffing. The goal is that agents find services that are genuinely appropriate for their tasks — not that Kindling-powered services dominate search results through volume or manipulation.

If a generated description is misleading or inflated, that is a bug. File an issue.

---

## Not first-party self-preferencing dressed as infrastructure

PUC operates services (b1e55ed, others) that participate in Kindling. These are labeled `is_first_party: true` everywhere they appear. They receive:

- No scoring adjustments
- No routing preference (first-party loses ties)
- No methodology exceptions

The first-party label is not a badge of honor — it's a disclosure of conflict. Judges should weight it accordingly.

---

## Not a protocol

Kindling is tooling built on top of protocols (x402, MCP, A2A, ERC-8004). It does not own or govern those protocols. Changes to those protocols may require changes to Kindling modules. Kindling participates in those ecosystems as a user and contributor, not a controller.

---

## Not permanent

If Kindling stops providing value, it should stop existing. The kill criteria in the spec are real. If Phase 1 targets aren't met:

> "Publish honest retrospective. Open-source everything as-is. Evaluate whether thesis is wrong or execution was wrong."

This is not a commitment to run forever. It's a commitment to honest accounting when things don't work.
